## Bugs / Risks
- **Impersonation magic link**: The `generateLink` API generates a link that could theoretically be intercepted if the redirect URL is leaked. Mitigated by: server-side only generation, single-use token, 30-min TTL, rate limited. The `hashed_token` is never exposed to the client — admin is redirected directly through `/auth/verify`.
- **Admin refresh token storage**: Stored in plaintext in `admin_impersonation_sessions.admin_refresh_token`. This is a sensitive credential. Protected by RLS (only admin can read own sessions) and service-role access only. Token expires naturally (Supabase JWT expiry).
- **`setSession` vs `refreshSession`**: Initially tried `setSession` with empty access_token — doesn't work. Fixed to use `refreshSession` which only needs the refresh token.
- **Non-async exports from `'use server'` files**: Turbopack rejects type re-exports and non-async function exports from `'use server'` modules. Solution: extract to a separate non-server `helpers.ts` file.
