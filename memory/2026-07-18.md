# memory/2026-07-18.md

## Features Shipped
- **AI Takeoff accuracy overhaul** (commit `e253aef`, development):
  - **Original image fix**: Client now fetches the pristine source image URL and sends it to the AI, instead of `canvas.toDataURL()` which included dark background (#1e293b), viewport transforms, zoom artifacts, and JPEG compression. This was the #1 root cause of poor detection.
  - **Two-phase prompt**: Phase 1 detects ONLY the roof outline (thick black perimeter). Phase 2 detects components (hips/valleys/ridges/barges/spouting) with the Phase 1 outline injected as context. Server runs both OpenAI calls and merges results.
  - **Angle snap fix**: `snapAndValidate` was applying 45° enforcement to barges and spouting (they fell through the `else` branch). Now barges/spouting correctly snap to 0°/90° (horizontal/vertical perimeter edges).
  - **Model upgrade**: `gpt-4o` → `gpt-4.1` (better spatial reasoning, still affordable for free tool).
  - **JPEG quality**: 90 → 95 on server-side preprocessing.
  - **Graceful degradation**: If Phase 2 (components) fails, returns Phase 1 (outline only) so user can still use the outline and draw components manually.

## Decisions
- Shaun confirmed: cannot afford GPT-5.6 for free tool scans. gpt-4.1 is the cost/quality compromise.
- Two-phase approach = 2 API calls per scan, but Phase 1 is simpler (fewer tokens) so cost increase is modest.
- Phase 1 outline is AUTHORITATIVE — Phase 2 components are mapped to Phase 1 areas, not the other way around.

## Changes
- `TakeoffWorkstation.tsx`: `handleAiScan` now fetches original image URL as blob → base64, sends that to API. Removed `canvas.toDataURL()` entirely.
- `ai-prompt.ts`: New `AI_TAKEOFF_PROMPT_PHASE1` (outline only) + `AI_TAKEOFF_PROMPT_PHASE2` (components with outline context). New `AI_TAKEOFF_PHASE1_SCHEMA`. Model default changed to `gpt-4.1`.
- `route.ts`: Two-phase call flow with `callVisionModel` helper. Merges Phase 1 + Phase 2 results.
- `applyAiResults.ts`: `snapAngle` and `snapToAngle` now handle barges/spouting as 0°/90°.

## Bugs / Risks
- Two API calls per scan = ~2x latency (expect 15-30s total). Acceptable for V1.
- Cost per scan roughly doubles (2 calls) but Phase 1 is simpler prompt → fewer output tokens.
- The `btoa()` call on large images could be slow on mobile — monitor.
