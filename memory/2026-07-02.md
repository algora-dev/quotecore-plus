## Bugs / Risks
- PowerShell mangled template literals (backticks) in FilesManager.tsx when writing via here-string — had to fix with a Node.js script. Lesson: avoid PowerShell here-strings with JS template literals; use `write` tool or Node scripts instead.
- `canvasReady` state was initially declared inside the canvas init effect scope but used in the reconstruction effect — had to hoist it to component-level state.
- Undo/redo on Takeoff canvas: pushHistorySnapshot calls are before mutations, but the exact mutation coverage should be verified during smoke testing (some edge cases like page switching may not be covered).
- Reconstruction relies on `canvas_points` being populated — old measurements (pre-this-build) won't have points and won't reconstruct. Only new measurements will reconstruct.
