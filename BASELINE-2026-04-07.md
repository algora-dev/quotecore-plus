## Lessons Learned
1. Always run `npm run build` (or `tsc --noEmit`) locally before deploying
2. Keep types.ts in sync with database schema changes
3. Dev mode (`npm run dev`) doesn't run TypeScript checking
4. Production strict mode catches errors dev mode doesn't
5. Use local TypeScript checking to find ALL errors at once (faster than Vercel loop)
6. Git tags + baseline branches = easy rollback safety net
